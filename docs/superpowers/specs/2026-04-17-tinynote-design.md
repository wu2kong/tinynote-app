# TinyNote 设计文档

## 1. 应用概述

TinyNote 是一款轻量级快捷笔记管理 + 快捷复制工具，旨在帮助用户高效管理和复制命令、代码片段和笔记。应用采用四栏布局，支持浅色和深色主题切换，提供简洁、现代、低打扰的工具型管理界面。

### 1.1 核心功能

- 笔记块管理：创建、编辑、删除、拖拽排序笔记块
- 快捷复制：一键复制笔记块内容到剪贴板
- 分组管理：支持无限层级的分组和笔记本
- 文件系统集成：直接映射到文件系统的目录和Markdown文件
- 主题切换：支持浅色和深色主题
- 视图切换：支持列表视图、卡片视图、紧凑视图
- 源码模式：支持直接编辑Markdown文件源码

## 2. 技术栈

- **前端框架**：ReactJS + TypeScript
- **桌面应用框架**：Tauri 2.0+
- **状态管理**：Zustand
- **文件操作**：Tauri fs API
- **剪贴板操作**：Tauri clipboard API
- **存储位置**：用户自定义目录，默认创建在用户选择的位置

## 3. 架构设计

### 3.1 整体架构

采用文件系统 + 内存缓存的方案，具体如下：

1. **文件系统层**：直接使用文件系统存储笔记文件，目录结构对应应用中的分组和笔记本
2. **缓存层**：在内存中维护笔记数据的缓存，提高操作性能
3. **应用层**：处理用户界面和交互逻辑，解析和维护特定格式的笔记文件

### 3.2 数据流向

1. 应用启动时，扫描用户选择的存储目录，构建内存缓存
2. 用户操作（如创建、编辑、删除笔记块）首先更新内存缓存
3. 定期或在用户手动保存时，将内存缓存同步到文件系统
4. 当文件系统发生外部变化时，重新扫描并更新内存缓存

## 4. 数据结构

### 4.1 笔记文件格式

采用 Markdown 扩展格式，使用特定分隔符区分笔记块：

```markdown
---
title: 笔记块标题
tags: [tag1, tag2]
---

笔记内容，可以是纯文本、JSON代码或各种语言的代码片段

---
title: 另一个笔记块标题
tags: [tag3]
---

另一个笔记块的内容
```

### 4.2 内存数据结构

```typescript
// 笔记块
interface NoteBlock {
  id: string;           // 唯一标识符
  title: string;        // 标题
  content: string;      // 内容
  tags: string[];       // 标签
  createdAt: string;    // 创建时间
  updatedAt: string;    // 更新时间
}

// 笔记本
interface Notebook {
  id: string;           // 唯一标识符
  name: string;         // 名称
  path: string;         // 文件路径
  noteBlocks: NoteBlock[]; // 笔记块列表
  isSourceMode: boolean; // 是否为源码模式
}

// 分组
interface Group {
  id: string;           // 唯一标识符
  name: string;         // 名称
  path: string;         // 目录路径
  children: (Group | Notebook)[]; // 子分组和笔记本
  notebookCount: number; // 笔记本数量（递归计算）
}

// 空间
interface Space {
  id: string;           // 唯一标识符
  name: string;         // 名称
  path: string;         // 目录路径（{空间名称}.tinynotes）
  groups: Group[];      // 分组列表
}

// 应用状态
interface AppState {
  spaces: Space[];      // 空间列表
  currentSpace: Space | null; // 当前空间
  currentGroup: Group | null; // 当前分组
  currentNotebook: Notebook | null; // 当前笔记本
  isDarkTheme: boolean; // 是否为深色主题
  viewMode: 'list' | 'card' | 'compact'; // 视图模式
  searchQuery: string; // 搜索查询
}
```

## 5. 界面设计

### 5.1 布局结构

采用四栏布局：

1. **应用栏**（最左侧-一个图标的宽度）：
   - 空间列表
   - 添加空间按钮
   - 主题切换入口
   - 设置入口

2. **目录栏**（左侧）：
   - 添加分组，点击分组展开/收起子分组和笔记本
   - 搜索框
   - 分组和笔记本列表（支持无限层级）

3. **笔记栏**（中间）：
   - 视图切换按钮（列表、卡片、紧凑）
   - 搜索框
   - 笔记块列表
   - 每个笔记块包含标题、内容预览和复制按钮
   - 新增笔记块按钮
   - 拖拽排序功能

4. **属性编辑栏**（右侧）：
   - 当点击笔记块时显示
   - 标题编辑
   - 内容编辑
   - 标签管理
   - 删除按钮

### 5.2 主题设计

- **浅色主题**：默认主题，适合明亮环境
- **深色主题**：适合夜间使用，减少眼睛疲劳

## 6. 功能实现

### 6.1 空间管理

- 创建空间：在用户选择的存储目录下创建 `{空间名称}.tinynotes` 目录
- 删除空间：删除对应的目录
- 切换空间：加载对应空间的分组和笔记本

### 6.2 分组和笔记本管理

- 创建分组：在当前分组下创建子目录
- 创建笔记本：在当前分组下创建 Markdown 文件
- 重命名：修改目录或文件名称
- 删除：删除目录或文件
- 搜索：根据名称搜索笔记本

### 6.3 笔记块管理

- 创建笔记块：在笔记本文件中添加新的笔记块
- 编辑笔记块：修改笔记块的标题、内容和标签
- 删除笔记块：从笔记本文件中移除笔记块
- 拖拽排序：调整笔记块的顺序
- 复制：点击复制按钮将笔记块内容复制到剪贴板

### 6.4 源码模式

- 切换到源码模式：直接编辑 Markdown 文件的原始内容
- 切换回笔记块模式：解析 Markdown 文件并以笔记块形式展示

### 6.5 主题切换

- 手动切换：通过设置菜单切换浅色/深色主题
- 自动切换：（可选）跟随系统主题设置

## 7. 存储方案

### 7.1 存储结构

```
{用户选择的存储位置}/
  ├── 空间1.tinynotes/
  │   ├── 分组1/
  │   │   ├── 子分组1/
  │   │   │   └── 笔记本1.md
  │   │   └── 笔记本2.md
  │   └── 分组2/
  │       └── 笔记本3.md
  └── 空间2.tinynotes/
      └── 笔记本4.md
```

### 7.2 缓存机制

- 应用启动时，扫描存储目录，构建内存缓存
- 提供重建缓存的入口，当数据损坏时使用
- 定期自动保存，确保数据持久化

### 7.3 数据同步

- 内存缓存与文件系统的双向同步
- 当检测到文件系统变化时，自动更新缓存
- 当用户修改笔记时，实时更新缓存并定期同步到文件系统

## 9. 技术挑战和解决方案

### 9.1 技术挑战

1. **文件系统操作性能**：频繁的文件读写可能影响性能
2. **笔记块解析**：需要高效解析和生成 Markdown 扩展格式
3. **拖拽排序**：实现流畅的拖拽排序体验
4. **主题切换**：确保所有组件正确响应主题变化

### 9.2 解决方案

1. **文件系统操作性能**：
   - 使用内存缓存减少文件系统操作
   - 批量写入和延迟写入策略
   - 异步操作避免阻塞 UI

2. **笔记块解析**：
   - 使用正则表达式高效解析 Markdown 扩展格式
   - 缓存解析结果，避免重复解析

3. **拖拽排序**：
   - 使用 React DnD 库实现拖拽功能
   - 优化拖拽过程中的视觉反馈

4. **主题切换**：
   - 使用 CSS Variables 实现主题变量
   - 全局状态管理主题设置
   - 确保所有组件使用主题变量而非硬编码颜色

## 10. 总结

TinyNote 是一款轻量级、高效的笔记管理和快捷复制工具，采用现代化的技术栈和架构设计。通过文件系统 + 内存缓存的方案，实现了高性能的笔记管理功能。四栏布局提供了清晰的信息层次，主题切换和多种视图模式满足了不同用户的使用习惯。

该设计方案充分考虑了用户需求和技术可行性，为后续的实现提供了清晰的指导。
